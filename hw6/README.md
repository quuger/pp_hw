assert сработал только на ARM с atomic.

На x86 есть гарантия на непереупорядовачивание записей, то есть строки `data = 42;` и `ready = true;` не переставляются => результат корректен.

А с atomic, посмотрев ассеблерный код для store, увидел реализацию через xchgb -- операцию, которая выполняется под lock, значит, переупорядочивания не будет.

На arm в ассеблере strb, то есть переупорядочивание может возникнуть, и воспризводится при большом количестве операций.

# Большое домашнее задание 1


## Протокол взаимодействия 

При установлении соединения с сервером, клиент отправляет сообщение `HELLO <hashrate>`

```
HELLO 123454321
```

Клиент принимает сообщения трех типов: `TASK`, `WAIT`, `DONE`. Тип сообщения определяется по первому слову.


### Формат сообщений
* `TASK <key> <left> <right>`

Выполняет поиск ключа `key` в диапазоне `[left, right]`

В случае успешного поиска вернется сообщение `FOUND <key>`, иначе `NOT_FOUND`

* `WAIT <time>`

Клиент засыпает на `time` секунд

* `DONE`

Клиент завершает работу


### Обработка ошибок ввода
* Если первое слово в сообщении не соответствует ни одному поддерживаемому типу, вернется `UNKNOWN_COMMAND`
* Если в сообщении недостаточно аргументов или невозможно их корректно интерпретировать, вернется `COMMAND_IS_ILL_FORMED`, возможно, с дополнительной диагностикой: `COMMAND_IS_ILL_FORMED: <error>` (см. [примеры](#примеры)).
* Лишние аргументы игнорируются

### Примеры
```
TASK e408a4e15e96e04693f077bce6d2d2043fe7a677 10 20  
FOUND 11
```

```
TASK e408a4e15e96e04693f077bce6d2d2043fe7a677 100 110
NOT_FOUND
```

```
UNKNOWN

```

```
TASK

```

```

```
